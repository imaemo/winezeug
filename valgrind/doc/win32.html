Finding memory bugs in Win32 applications with Valgrind

Dynamic Analysis

Writing good software is hard. Programmers must always be on the lookout for bugs. One of the many techniques developed to help programmers do this is called dynamic analysis. In dynamic analysis, programs are run in a special environment that watches for flaws such as invalid pointers, undefined values, or race conditions.

The classic dynamic analysis tool is Purify. It's good, but expensive. The free software world developed a similar tool, Valgrind, which is also good, but does not run on Windows.

However, Linux can run Windows apps via the Wine compatibility layer, and Valgrind is compatible with Wine, so Valgrind could potentially replace some of the uses of Purify.

Chromium, Valgrind, and Win32

When the Chromium project needed to add dynamic analysis to its continuous build and test rig, it used Purify on Win32, and Valgrind on Linux and Mac. To investigate whether Chromium could use Valgrind for Win32, I wrote two scripts: one to package up the Chromium unit tests built for Windows so they could be copied to a Linux system, and another to run the Chromium unit tests under Valgrind.
